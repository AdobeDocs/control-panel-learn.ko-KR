# 문서 참여

Adobe 커뮤니티와 설명서 팀 외부 Adobe 직원의 참여도 환영합니다.

## Adobe 오픈 소스 행동 강령

이 프로젝트는 [Adobe 오픈 소스 행동 강령](code-of-conduct.md) 또는 [.NET Foundation 행동 강령](https://dotnetfoundation.org/code-of-conduct)을 채택했습니다. 자세한 내용은 [참여](contributing.md) 문서를 참조하십시오.

## Adobe 콘텐츠 참여 관련 정보

[Adobe 문서 참여자 안내서](https://experienceleague.adobe.com/docs/contributor/contributor-guide/introduction.html?lang=ko)를 참조하십시오.

참여 방식은 참여자와 참여자가 변경하려는 사항의 종류에 따라 다릅니다.

### 사소한 변경 사항

선의로 사소한 업데이트에 참여하려는 경우, 문서를 방문하여 문서의 **편집** 링크를 클릭하면 문서의 GitHub 소스로 이동하게 됩니다. 그 뒤에는 간단히 GitHub UI를 사용하여 업데이트를 적용할 수 있습니다. 자세한 내용은 일반 [Adobe 문서 참여자 안내서](https://experienceleague.adobe.com/docs/contributor/contributor-guide/introduction.html?lang=ko)를 참조하십시오.

이 저장소의 설명서 및 코드 예제에 대하여 제출하는 소규모 정정이나 추가 설명에는 Adobe 사용 약관이 적용됩니다.

### 커뮤니티 멤버의 큰 변경 사항 또는 새 문서

Adobe 커뮤니티에 소속되어 있고 새 문서를 만들거나 큰 변경 사항을 제출하려는 경우 Git 저장소의 문제 탭을 사용하여 문제를 제출하면 설명서 팀과 대화를 시작할 수 있습니다. 변경/작성 계획에 동의하면 직원과의 협력을 통해 공개 및 비공개 저장소의 작업을 결합하여 새로운 콘텐츠를 제공할 수 있습니다.

### Adobe 직원의 큰 변경 사항

Adobe Experience Cloud 솔루션 제품 팀의 테크니컬 라이터, 프로그램 관리자, 개발자 중 하나인 동시에 기술 문서 기여자/작성자인 직원은 비공개 저장소(`https://git.corp.adobe.com/AdobeDocs`)를 사용해야 합니다.

## 도구 및 설정

커뮤니티 참여자는 GitHub UI를 사용해 기본적인 편집을 수행하거나 저장소를 포크해 대규모 변경/작성을 수행할 수 있습니다.

자세한 내용은 [Adobe 문서 참여자 안내서](https://experienceleague.adobe.com/docs/contributor/contributor-guide/introduction.html?lang=ko)를 참조하십시오.

## Markdown을 사용하여 주제 서식을 지정하는 방법

이 저장소의 모든 문서는 GitHub 버전의 Markdown을 사용합니다. Markdown에 익숙하지 않은 경우 다음을 참조하십시오.

* [Markdown 기본 사항](https://help.github.com/articles/getting-started-with-writing-and-formatting-on-github/)
* [인쇄 가능한 Markdown 치트 시트](https://guides.github.com/pdfs/markdown-cheatsheet-online.pdf)

## 레이블

공개 저장소에서는 당사가 끌어오기 요청 워크플로를 관리하고, 끌어오기 요청 진행 상황을 사용자에게 알리는 과정을 지원하기 위해 끌어오기 요청에 자동 레이블이 지정됩니다.

* **변경 내용이 작성자에게 보내짐**: 작성자가 보류 중인 끌어오기 요청에 대한 알림을 받았습니다.
* **병합 준비 완료**: 끌어오기 요청 검토 팀에서 검토할 준비가 되었습니다.
